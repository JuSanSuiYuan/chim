# Chim （中文名：启语）语言语法规则（最新版）

## 设计目标
- 简洁与一致：对标 V 语言的简洁风格，去除冗余符号
- 缩进即结构：采用 Python/Nim 风格的缩进式代码块，不使用花括号 `{}`，不使用 `end`
- 语义清晰：统一字面量构造、唯一分支模型、明确的类型推断
- 中英关键词双轨：保留中文关键词，同时支持精炼英文别名（`fn/struct/enum/import/export/pub/mut/match/case/default/return`）

## 词法与缩进
- 行驱动词法：按行扫描，自动注入 `INDENT`/`DEDENT`/`NEWLINE` 标记以表示代码块结构
- 注释：`# ...` 或 `// ...` 单行注释；多行注释暂不推荐
- 关键字（中文/英文别名）：
  - 定义：`函数`/`fn`，`结构体`/`struct`，`枚举`/`enum`
  - 模块：`导入`/`import`，`导出`/`export`，`公共`/`pub`
  - 变量：`可变`/`mut`，`设`/`var`，`令`/`let`，
  - 分支：`匹配`/`match`，`案例`/`case`，`默认`/`_`，`如果`/`if`，`否则如果`/`elif`，`否则`/`else`（仅 `兼容` 方言）
  - 方言：`安全`/`safe`，`系统`/`system`，`兼容`/`compat`，`动态`/`dynamic`，`DSL`/`toml`
  - 不安全块：`不安全`/`unsafe`
  - 循环：`对于`/`for`，`在`/`in`，`跳出`/`break`，`继续`/`continue`，`当`/`while`（仅 `兼容` 方言）
  - 并发：`通道`/`chan`
  - 其他：`返回`/`return`，`外部`/`extern`
 - 运算符与符号：`+ - * / % ** == != >= <= > < ! && || = := : , . .. ... :: -> () []`
   - 说明：`...` 为闭区间范围，`..` 为半开区间范围；`::` 为模块/命名空间路径分隔；`<-` 为通道发送/接收运算符（见“并发与通道”）
- 花括号 `{}` 与 `end`：不再接受；请使用冒号 `:`+缩进代码块

## 代码块与语句
- 头部使用冒号 `:`，块体以缩进表示，结束通过 `DEDENT`
- 语句以 `NEWLINE` 分隔；不需要行尾分号
- 示例：
  ```chim
  fn 初始化():
      配置()
      启动()
  ```

## 变量声明与类型推断
- 声明引导词：
  - `令` / `let`：不可变绑定
  - `设` / `var`：可变绑定
- 简洁声明：
  - `令 name := 表达式`
  - `设 name := 表达式`
- 带类型声明：
  - `令 name: 类型 = 表达式`（也可写作 `令 name: 类型 := 表达式`）
  - `设 name: 类型 = 表达式`（也可写作 `设 name: 类型 := 表达式`）
- 类型占位 `_`：
  - `令 name: _ = 表达式` 或 `设 name: _ = 表达式` 表示由初始化表达式推断类型
- 赋值与重绑定：
  - 仅 `设/var` 允许后续使用 `=` 进行值更新；`令/let` 禁止重新赋值
- 兼容方言（compat）：
  - `mut name := 表达式` 等价于 `设 name := 表达式`
  - `name := 表达式` 默认为 `令/let`；推荐显式书写 `令/设`

示例：
```chim
令 a := 1
设 b: 整数 = 2
设 b = b + a
令 c: _ = a + b
```

## 函数定义
- 语法：`fn 名称(参数[, ...]) -> 返回类型:` 后接缩进块
- 返回语句：`返回 表达式` 或省略（返回单元类型）
- 示例：
  ```chim
  fn 加法(a: 整数, b: 整数) -> 整数:
      返回 a + b
  ```

## 方言与安全模型
- 文件级方言声明（置于文件首行，可选）：
  - `安全`（或 `safe`）：默认安全模式；禁止使用不安全代码块
  - `系统`（或 `system`）：系统方言；允许不安全块与零拷贝接口
- 不安全块（仅在系统方言）：
  - 语法：`不安全:` 后接缩进代码块
  - 行为：语义层将对块内语句进行类型检查，但在安全方言中出现将报错

## 唯一分支模型：match
- 语法（Swift风格增强）：
  ```chim
  匹配 表达式:
      案例 模式 ->
          块
      案例 模式 当 条件 ->  # guard条件
          块
      ...
      _ ->
          块
  
  ```
- 模式类型：
  - 值匹配：`案例 1 ->` 匹配字面量值
  - 元组模式：`案例 (0, 0) ->` 匹配元组结构
  - 范围模式：`案例 90..100 ->` 匹配数值范围（半开区间）
  - 范围模式：`案例 90...100 ->` 匹配数值范围（闭区间）
  - 模式变量：`案例 (x, 0) ->` 绑定变量，任意匹配该位置
  - Guard条件：`案例 x 当 x > 0 ->` 附加条件判断
- 支持布尔表达式：`匹配 (条件): 案例 真 -> ... _ -> ... `
- 强制规则（核心语法）：仅支持 `match`；`if/else/elif/switch` 不属于核心语法
- 兼容方言：在文件首行声明 `兼容`/`compat` 时，允许 `if/elif/else` 作为语法糖，等价降解为嵌套 `匹配` 结构；例如：
  - `if cond: A elif cond2: B else: C` → `匹配 (cond): 案例 真 -> A _ -> 匹配 (cond2): 案例 真 -> B _ -> C`
- 迁移示例：
  - `if x > 10: A else: B` → `匹配 (x > 10): 案例 真 -> A _ -> B`
  - `elif` 链 → 以嵌套或多分支 `案例` 表达
  - 所有条件判断统一为 `匹配 (布尔表达式)` 结构
- 完整示例：
  ```chim
  # 基础值匹配
  匹配 数字:
      案例 1 -> 输出("一")
      案例 2 -> 输出("二")
      _ -> 输出("其他")
  
  # 元组模式匹配
  匹配 点:
      案例 (0, 0) -> 输出("原点")
      案例 (x, 0) -> 输出("在X轴")
      案例 (0, y) -> 输出("在Y轴")
      _ -> 输出("其他位置")
  
  # 范围匹配
  匹配 分数:
      案例 90..100 -> 输出("优秀")
      案例 80..90 -> 输出("良好")
      案例 60..80 -> 输出("及格")
      _ -> 输出("不及格")
  
  # Guard条件（Swift风格）
  匹配 值:
      案例 x 当 x > 0 -> 输出("正数")
      案例 x 当 x < 0 -> 输出("负数")
      _ -> 输出("零")
  ```

## 循环与递归
- 已移除：`while`（核心语法）
- 正式循环语法：
  - 计数/迭代：`对于 变量 在 表达式:` 或 `for name in expr:`
    - 例：`对于 i 在 0..n:`、`for (k, v) in 映射(...):`
  - 跳转控制：`跳出`/`break`，`继续`/`continue`
- 兼容方言循环：
  - 条件循环：`当 条件:` 或 `while 条件:` 后接缩进块；可使用 `break`/`continue`
  - 语义：等价于每次迭代先评估条件，条件为真则执行循环体，否则结束；编译期可降解为显式条件跳转
- 推荐：递归在纯函数场景可用；长迭代场景建议使用 `对于/for`，当前不保证尾调用优化
- 示例（递归计数）：
  ```chim
  fn 递归计数(c: 整数):
      匹配 c:
          案例 5 -> 返回
          _ ->
              输出("计数:", c)
              递归计数(c + 1)
  ```

## 结构体与字面量构造（统一括号）
- 结构体初始化：`类型名(字段: 表达式[, ...])`
  - 示例：`Person(name: "张三", age: 25, is_active: 真)`
- 统一容器构造：
  - 数组：`数组( expr1, expr2, ... )` 或 `array(...)`
  - 集合：`集合( ... )` 或 `set( ... )`
  - 元组：`元组( ... )` 或 `tuple( ... )`
  - 映射：`映射<K,V>(key: value, ...)` 或 `map<K,V>(key: value, ...)`
- 类型推断（简化规则）：
  - 数组/集合：按首元素推断元素类型（不一致时退化为 `Any`）
  - 元组：逐元素类型组成元组类型
  - 映射：若未显式给出 `<K,V>`，按首个键值对推断 `K` 与 `V`

## 表达式与操作
- 标识符、字面量（整数/浮点/字符/字符串/布尔/空）
- 一元/二元运算：与词法表一致
- 成员访问：`目标.成员`
- 索引访问：`目标[索引]`
- 调用：`函数名(实参...)`
- 构造消歧规则：以类型名（建议首字母大写）+命名参数表示结构体/映射构造；以普通标识符+位置实参表示函数调用

## 类型系统
- 基本类型：`整数`、`浮点数`、`字符`、`字符串`、`布尔`、`单元`
- 复合类型：`数组<T>`、`引用<T>`（可变 `mut 引用<T>`）、`函数(参数类型) -> 返回类型`、`元组(T1, T2, ...)`
- 命名类型：`类型 别名 = 实际类型`

## 并发与通道
- 通道类型：`通道<T>` / `chan<T>`
- 方向化通道类型：
  - 仅发送：`chan<- T`
  - 仅接收：`<-chan T`
- 创建：`chan<T>(容量)` 返回有界通道；`chan<T>()` 为无界或实现定义容量
- 发送：`ch <- x` 将 `x` 发送到通道 `ch`
- 接收：
  - 声明接收：`令 v := <- ch` 或 `设 v := <- ch`
  - 赋值接收：`设 v = <- ch`
- 方向约束：
  - `chan<- T` 仅允许发送；`<-chan T` 仅允许接收
  - 违反方向的操作为语义错误
- 与分支的关系：
  - 选择性并发可通过库函数或专用构造实现；推荐使用 `匹配` 搭配非阻塞试探 API，而不引入通用分支关键字
- 示例：
```chim
令 ch := chan<整数>(16)
ch <- 1
设 x := <- ch
```

## 模块与可见性
- 模块声明（文件首行可选）：`模块 名称` / `module name`
- 导入：`导入 路径[ 作为 别名]` / `import path[ as alias]`，模块路径使用 `::` 分隔
- 导出：`导出 名称` / `export name`
- 可见性：`公共` / `pub`（用于对外暴露）

## 组生命周期与所有权（系统方言）
- 组块语法：`组 名称:` 后接缩进块；仅在 `系统` 方言中可用
- 组内分配：在组块内创建的资源归属该组，块结束时按组规则统一回收
- 快照与句柄：
  - `快照`：只读、可跨组传递，不延长源组生命周期（等价 `system::snapshot(x)`）
  - `句柄`：轻量引用、可跨边界传递，行为受宿主约定（等价 `system::handle(x)`）
- 引用规则：
  - 只读引用 `引用<T>` 与可写引用 `mut 引用<T>` 仅在组内合法，跨组需通过快照/句柄
  - 组外不可持有指向组内可写资源的直接引用
- 移动与释放：
  - 资源默认按“移动语义”传递；跨组需要显式快照/句柄
  - 组结束自动释放归属资源；快照/句柄的释放按宿主运行时策略
- 语义约束（前端）：
  - 在 `系统` 方言下，前端拒绝组外直接持有组内可写引用
  - 允许在 `兼容` 方言中忽略上述检查（迁移模式）
## 运行时与外部接口（示例约定）
- 外部函数：`extern fn 名称(参数...) -> 类型:` 块体由宿主实现；`extern` 为正式关键字
- 特权注解（NovaOS 场景）：`特权函数<级别>`（语义按宿主约定）
- system 标准库占位（零拷贝与组模型协同）：
  - `system::arena()`：返回批量分配器，占位用于编译期优化布局与回收
  - `system::snapshot(x)`：生成只读快照，可跨组/边界传递且不延长源组寿命
  - `system::handle(x)`：生成轻量句柄，用于跨边界引用占位

## 原始缓冲区与延后释放（系统方言，预览）
- 关键字：`原始` / `raw`，`延后` / `defer`
- 原始缓冲区：
  - 目的：在 `不安全` 块中进行低级内存访问与批量数据操作
  - 容器类型：`原始<T>` / `raw<T>`（仅在系统方言）
  - 构造：预留语法 `raw[...]` / `原始[...]` 或 `原始<T>(容量)`；当前实现以组分配与数组构造替代，正式语法将在后续版本提供
  - 访问：索引与结构体成员访问仅允许在 `不安全` 块中，需遵守边界与对齐检查
  - 调整：预留 `resize` 能力以修改缓冲区容量；与组生命周期协同（缓冲区归属某组）
- 延后释放：
  - 语法：`延后 资源 调用` / `defer 资源 调用`（例如：`延后 文件 关闭()`）
  - 语义：注册作用域退出时的清理操作；与组结束释放互补，用于非内存资源（文件/句柄）与自定义清理逻辑
  - 限制：在安全方言中仅允许无副作用的清理；系统方言允许与不安全块协作
- 与组模型的协同：
  - 原始缓冲区与延后释放均受组生命周期约束；组结束先执行已注册的延后清理，再统一释放组内资源
  - 跨组传递时仅允许快照/句柄，不允许直接共享可写原始缓冲区

## 动态方言（预览）
- 概述：`动态`/`dynamic` 文件级方言启用更宽松的类型与调度语义，用于原型与脚本式编程；语法不变，语义放宽
- 类型与调度：
  - 默认动态类型：未注解的变量与形参默认为 `Any`
  - 多分派：按实参运行时类型选择最匹配的函数版本；无匹配时报错或回退至 `Any` 版本
  - 反射：允许运行时查询类型名与成员列表（受宿主运行时支持）
- 约束：
  - 与 `安全` 方言互斥；可与 `系统` 叠加但会降低静态检查强度
  - 仍保留 `match` 为核心分支；兼容方言的 `if/while` 可作为语法糖
- 适用场景：快速脚本、REPL、DSL宿主集成、原型验证

## DSL（TOML）方言（预览）
- 概述：`DSL`/`toml` 文件级方言将文件视为TOML风格的领域配置而非可执行源代码
- 语义：
  - 键/值对映射为语言或工具链可消费的配置对象（如 `package.chim` 的依赖管理）
  - 仅允许TOML合法构造；不解析函数/结构体/语句；编译器前端将此方言交给DSL解析器
- 互操作：
  - 与 `chim mod` 包管理器协同，作为项目清单/锁文件的数据来源
  - 可通过导入机制或工具链API将DSL数据注入可执行代码
- 适用场景：包管理、构建配置、元数据声明、嵌入式领域描述

## 不兼容项与迁移
- 核心语法不再支持：`{}` 字面量、`end`、`while`
- 迁移策略：
  - 结构体 `Type{ a:1 }` → `Type(a: 1)`
  - 条件：
    - 核心语法：`if/else` 迁移为 `匹配 (cond) | 案例 真 -> ... _ -> ...`
    - 兼容方言：保持 `if/else`，按语法糖等价生成
  - 循环：`while cond ...` → 使用 `对于/for in` 或递归

### 迁移示例（旧 → 新/兼容）
- 条件分支：
```chim
# 旧
if x > 10:
    A()
else:
    B()

# 新（核心）
匹配 (x > 10):
    案例 真 -> A()
    _ -> B()

# 兼容（语法糖，等价降解）
if x > 10:
    A()
else:
    B()
```

- 多分支：
```chim
# 旧
if a == 1:
    P()
elif a == 2:
    Q()
else:
    R()

# 新（核心，直接使用match多案例）
匹配 a:
    案例 1 -> P()
    案例 2 -> Q()
    _ -> R()

# 或嵌套布尔匹配
匹配 (a == 1):
    案例 真 -> P()
    _ -> 匹配 (a == 2):
        案例 真 -> Q()
        _ -> R()
```

- 循环（while）：
```chim
# 旧
while i < n:
    work(i)
    i = i + 1

# 新（核心）
对于 i 在 0..n:
    work(i)

# 兼容（允许当/while）
设 i := 0
while i < n:
    work(i)
    设 i = i + 1
```

- 结构体字面量：
```chim
# 旧
Point{ x: 1.0, y: 2.0 }

# 新
Point(x: 1.0, y: 2.0)
```

- 变量声明与可变性：
```chim
# 旧
mut count := 0
value := 1

# 新（核心）
设 count := 0
令 value := 1

# 兼容（语法糖）
mut count := 0    # 等价 设 count := 0
value := 1        # 默认为 令 value := 1
```

- 构造与调用消歧：
```chim
# 构造（类型名 + 命名参数）
Person(name: "张三", age: 25)

# 调用（普通标识符 + 位置实参）
make_person("张三", 25)
```

- 模块与导入：
```chim
# 新
模块 示例
导入 system::io 作为 io
公共 fn 主():
    io.输出("hello")
```

- 容器构造：
```chim
数组(1, 2, 3)
集合(1, 2, 3)
映射<字符串, 整数>(key: "a", value: 1)
```

## 下划线 `_` 的语义约束
- 模式位：在 `匹配` 中，`_` 表示通配模式（默认分支）
- 类型位：在变量声明中，`name: _ = 表达式` 或 `name: _ := 表达式` 表示由初始化表达式推断类型
- 表达式位：`_ := 表达式` 或 `_ = 表达式` 表示显式丢弃结果；仅允许在语句位置使用
- 以上三种用法按语境消歧，禁止交叉混用（例如在类型位或表达式位使用模式 `_`）

## 缩进与续行细则
- 仅在以冒号 `:` 结束的行且下一行缩进增加时注入 `INDENT`
- 空行与仅注释行不改变缩进栈
- 括号未闭合的行允许续行；也可使用反斜杠续行（可选）

## 范围与迭代约定
- `a...b` 表示闭区间，包含 `a` 与 `b`
- `a..b` 表示半开区间，包含 `a` 不包含 `b`
- 迭代示例：`对于 i 在 0..n:`、`对于 x 在 集合(...):`

## 兼容与方言
- 文件首行可声明：`安全` / `系统` / `兼容`
- `兼容` 方言允许：
  - 使用 `if/elif/else` 语法糖（等价生成 `match`）
  - 保留旧代码中的部分符号，但不接受 `{}` 与 `end`
  - 循环使用 `对于/for in`，并允许 `当/while` 条件循环（语义等价降解）


## 代码示例（综合）
```chim
system

struct 点:
    x: 浮点数
    y: 浮点数

fn 距离(a: 点, b: 点) -> 浮点数:
    dx := a.x - b.x
    dy := a.y - b.y
    返回 (dx * dx + dy * dy)

fn 主():
    a := system::arena()
    p1 := 点(x: 1.0, y: 2.0)
    p2 := 点(x: 4.0, y: 6.0)
    d := 距离(p1, p2)
    匹配 (d > 5.0):
        案例 真 -> 输出("远")
        _ -> 输出("近")
    arr := 数组(1, 2, 3)
    m := 映射(key: "value", count: 3)
    s := system::snapshot(arr)
    不安全:
        _ := system::write(arr, arr)
    递归计数(0)
```

## 实现状态与注意
- 前端解析：已实现缩进块、`match`、括号字面量、`:=` 变量、文件级方言与不安全块
- 语义推断：数组/集合/元组/映射的基础推断已就绪；映射类型为占位；在安全方言中拒绝不安全块
- 并发：`chan<T>` 与 `<-` 作为通道并发原语纳入规范
- IR/后端：容器构造与组分配的代码生成将结合 `system::arena/snapshot/handle` 做优化
- 建议：统一项目中所有旧风格（`{}`、`end`、`if/while`）为上述语法；按需选择 `安全/系统/兼容` 方言

## 风格与约定
- 类型名建议使用首字母大写；函数/变量使用小写
- 优先显式书写 `令/设` 指示可变性
- 构造使用命名参数；函数调用使用位置实参
- 禁止行尾分号；统一缩进风格并避免混合制表符/空格
- 默认分支统一书写为 `_ ->`；`默认 ->` 作为别名允许，格式化将归一化为 `_ ->`
